# 版本冻结 + 上线准备说明

## 版本信息
- **版本号**: v1.0.0
- **冻结时间**: 2026-06-18
- **状态**: 版本冻结，仅允许修复阻断性bug

## 代码冻结规则

### ✅ 允许的操作
- 修复阻断性bug（系统崩溃、核心功能不可用）
- 安全漏洞修复
- 性能优化（不改变功能逻辑）
- 文档更新

### ❌ 禁止的操作
- 新增功能
- 修改AI调用逻辑
- 重构代码结构
- 修改UI/UX
- 更新依赖包（除非安全修复）

## 环境模式配置

### DEV模式（开发环境）
- **强制使用Mock**：禁止调用真实AI API
- **限流开启**：单用户每分钟3次
- **日志开启**：详细日志输出
- **指标关闭**：不收集性能指标

### PROD模式（生产环境）
- **允许真实AI**：调用火山方舟API
- **限流开启**：单用户每分钟3次
- **日志开启**：关键日志输出
- **指标开启**：收集性能指标

### TEST模式（测试环境）
- **强制使用Mock**：禁止调用真实AI API
- **限流关闭**：无限制
- **日志开启**：详细日志输出
- **指标关闭**：不收集性能指标

## 成本控制措施

### 1. API调用限制
- DEV模式：完全禁用真实API调用
- PROD模式：单用户每分钟最多3次
- 自动fallback到Mock模式

### 2. 任务去重
- 30秒内相同任务自动去重
- 防止重复点击触发多次API调用
- 基于用户ID + 提示词 + 图片URL去重

### 3. 限流机制
- 基于用户IP或Token识别
- 滑动窗口算法
- 返回429状态码 + Retry-After头

## 紧急修复流程

### 1. 发现阻断性bug
- 记录bug详情
- 评估影响范围
- 确定修复优先级

### 2. 修复流程
- 创建修复分支
- 最小化修改范围
- 充分测试修复
- 合并到主分支

### 3. 部署流程
- 通知相关人员
- 选择低峰时段部署
- 验证修复效果
- 监控系统状态
