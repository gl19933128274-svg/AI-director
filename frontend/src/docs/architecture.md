## 系统分层说明

### 1. 用户层 (User Layer)
- **Web 前端**: 基于 Next.js 的 React 应用，提供可视化创作界面
- **移动端**: 预留移动端适配能力
- **API 调用者**: 支持第三方集成的 RESTful API
- **管理后台**: 系统管理、监控面板、配置管理

### 2. API Gateway 层
- **统一入口**: 所有外部请求通过 Next.js API Routes 进入
- **路由分发**: 根据路径分发到不同业务服务
- **请求追踪**: 每个请求携带 request_id, user_id, trace_id
- **安全控制**: 认证、限流、请求校验

### 3. 核心业务服务层

#### Release Control (灰度发布系统)
| 功能 | 说明 |
|------|------|
| 用户分组 | 支持 100%/10%/1% 流量控制 |
| 流量分配 | 基于 user_id Hash 算法分流 |
| 功能开关 | video_generation_enabled, scene_generation_enabled |
| Kill Switch | 一键紧急关闭所有 AI 调用 |
| 白名单/黑名单 | 支持特定用户豁免或封禁 |

#### Cost Control (成本监控系统)
| 功能 | 说明 |
|------|------|
| 成本记录 | 记录每次 AI 调用的完整成本信息 |
| 用户成本统计 | `/api/cost/user` - 按用户查询成本 |
| 每日成本统计 | `/api/cost/daily` - 按日期查询成本 |
| 项目成本统计 | `/api/cost/project` - 按项目查询成本 |
| 成本上限控制 | 单用户每日/每月最大成本限制 |
| Mock Fallback | 超限后自动切换到 Mock 模式 |

#### Monitoring (可视化监控系统)
| 指标 | 说明 |
|------|------|
| 请求成功率 | 整体 API 请求成功比例 |
| AI 调用成功率 | AI 服务调用成功比例 |
| 平均延迟 | 请求平均响应时间 |
| P95/P99 延迟 | 长尾延迟统计 |
| 错误分布 | AI_ERROR, AUTH_ERROR, STORAGE_ERROR, LIMIT_ERROR |
| 状态码分布 | 401/403/500 等状态码计数 |
| 统一日志 | 结构化日志，支持 request_id 追踪 |

### 4. 任务队列层
- **异步处理**: 视频生成等耗时任务异步执行
- **状态管理**: pending → processing → completed/failed
- **轮询机制**: 定时轮询 AI 服务获取任务状态

### 5. AI 服务层
- **主服务**: 火山引擎 Ark API (doubao-seedance-1-5-pro)
- **备选服务**: Replicate, Runway (Fallback)
- **模型适配**: 统一适配器模式，支持多模型切换

### 6. 存储层
- **对象存储**: 火山引擎 TOS，存储视频、图片文件
- **关系数据库**: PostgreSQL，存储任务、项目、用户数据

## 错误分类体系

| 错误类型 | 代码前缀 | 说明 |
|----------|----------|------|
| AI_ERROR | AI_ | AI 服务调用失败 |
| AUTH_ERROR | AUTH_ | 认证失败 |
| STORAGE_ERROR | STORAGE_ | 存储操作失败 |
| LIMIT_ERROR | LIMIT_ | 限流/成本超限 |
| VALIDATION_ERROR | VALIDATION_ | 请求参数校验失败 |
| TIMEOUT_ERROR | TIMEOUT_ | 请求超时 |
| SERVER_ERROR | SERVER_ | 服务端错误 |

## 关键设计原则

### 1. 可控成本
- 实时成本记录与统计
- 多维度成本监控（用户/项目/时间）
- 成本超限自动降级（Mock 模式）

### 2. 可灰度发布
- 基于用户 Hash 的流量控制
- 灵活的功能开关
- 一键 Kill Switch

### 3. 可观测
- 全链路追踪（request_id, user_id, trace_id）
- 统一日志格式
- 多维度监控指标

### 4. 可快速定位问题
- 结构化日志便于搜索
- 错误分类体系
- 完整的请求链路记录

### 5. 可上线生产环境
- 完整的异常处理机制
- 降级与容错策略
- 性能监控与告警能力
