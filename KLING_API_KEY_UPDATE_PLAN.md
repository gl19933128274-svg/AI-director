## 🔍 问题分析

### 当前状态
- **错误代码**: HTTP 401, Code: 1003
- **错误信息**: "Authorization is not active"
- **根本原因**: 当前 API Key/Secret Key 无效或未激活

### 影响范围
- ✅ 视频生成功能不可用
- ✅ Fallback 机制已生效（自动切换到混元）

## 📝 更新步骤

### 阶段 1: 准备工作（5分钟）

#### 1.1 备份当前配置
```bash
# 备份当前环境变量配置
cp frontend/.env.local frontend/.env.local.backup.$(date +%Y%m%d)
```

#### 1.2 准备新的 API Key
登录可灵开发者平台：https://klingai.kuaishou.com/dev

| 步骤 | 操作 |
|------|------|
| 1 | 登录账号 |
| 2 | 进入 API 管理页面 |
| 3 | 生成新的 Access Key 和 Secret Key |
| 4 | 确保 Key 已激活 |
| 5 | 记录新的 Key（安全保存） |

#### 1.3 更新环境变量
编辑 `frontend/.env.local`：

```env
# 可灵 AI - 更新后的密钥
KLING_API_KEY=YOUR_NEW_ACCESS_KEY
KLING_SECRET_KEY=YOUR_NEW_SECRET_KEY
KLING_API_BASE=https://api-beijing.klingai.com

# 启用调试模式（可选，测试完成后关闭）
KLING_DEBUG=true
```

## 🛡️ 回滚方案

### 紧急回滚步骤

#### 方案 A: 快速回滚（5分钟）
```bash
# 恢复备份的配置
cp frontend/.env.local.backup.* frontend/.env.local

# 重启服务
npm run build
npm run start
```

#### 方案 B: 功能降级（立即）
```bash
# 临时禁用可灵 API，仅使用本地 mock
# 更新 .env.local
USE_REAL_AI=false
```

#### 方案 C: Fallback 机制（自动）
系统已内置 fallback 机制：
- 可灵失败 → 自动切换到混元（如果适用）
- 用户无感知降级

### 回滚触发条件

| 条件 | 操作 |
|------|------|
| API 调用失败率 > 5% | 立即回滚 |
| 响应时间 > 5秒 | 立即回滚 |
| 用户投诉 | 评估后回滚 |
| 系统异常告警 | 立即回滚 |
